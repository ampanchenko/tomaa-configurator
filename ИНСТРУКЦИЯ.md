# TOMAA — онлайн-конфигуратор браслета

Интерактивный конструктор: посетитель выбирает **основу** — геометрию (узкий 10мм / широкий
15мм, один оборот / на два оборота), **материал** (кожа/бархат) и **цвет** (5–6 цветов) — и
**надевает до 5 шармов** в ряд. Каждый шарм со своим **покрытием** (золото/серебро), их можно
**смешивать**. Клик по шарму в списке — надевает на браслет; клик по шарму на браслете —
снимает; перетаскивание мышкой — меняет местами. Для основ на два оборота шармы надеваются на
выбранный **уровень** (верхний/нижний, до 5 на каждый). Цена и состав считаются по реальному
каталогу Tilda; кнопка «Оформить заказ» добавляет основу и шармы в корзину Tilda. Сделан одним
самодостаточным файлом `index.html` (HTML+CSS+JS, без сторонних библиотек) — удобно для Tilda.

> Цвета браслетов — программная перекраска чёрной основы по яркости (`recolor_bases.py`),
> поэтому для светлых кож и бархата цвет приблизительный. Формы 15мм и «двух оборотов» собраны
> из фото 10мм — для точности нужны настоящие развёрнутые фото этих основ.

## 1. Что внутри папки

```
Конфигуратор/
├── index.html            ← сам конфигуратор (открывается в браузере)
├── tilda-embed.html      ← готовый код для вставки на сайт (Tilda)
├── ИНСТРУКЦИЯ.md         ← этот файл
├── assets/               ← оптимизированные картинки для веба (~1 МБ)
│   ├── bracelets/        ← основы: single-10, single-15, double-10, double-15 (.webp)
│   └── charms/
│       ├── gold/         ← шармы для наложения, золото (700×700)
│       ├── silver/       ← шармы для наложения, серебро
│       └── preview/      ← мелкие иконки шармов для кнопок выбора
└── images/               ← ИСХОДНИКИ (оригиналы), в вебе не используются
```

> `assets/` создаётся из `images/` автоматически и оптимизируется (WebP, ~1 МБ вместо 17 МБ).
> Папку `images/` можно держать как архив оригиналов — на сайт она не нужна.

---

## 2. Как посмотреть на компьютере

Просто двойным кликом по `index.html` — откроется в браузере. Если шармы не наложатся
(некоторые браузеры блокируют локальные файлы), запустите локальный сервер из папки:

```
python3 -m http.server 8000
```
и откройте `http://localhost:8000`.

## 3. Как разместить на tomaa.ru (Tilda)

Сайт на Tilda не умеет хранить целую папку с файлами, поэтому конфигуратор нужно сначала
**выложить по интернет-адресу**, а потом вставить на страницу через `iframe`. Это надёжный
способ: верстка Tilda и конфигуратор не мешают друг другу.

### Шаг 1. Выложить папку в интернет
Подойдёт любой статический хостинг. Самые простые:
- **Netlify Drop** — netlify.com/drop: перетащите папку `Конфигуратор` в окно, получите
  адрес вида `https://tomaa-cfg.netlify.app`. Бесплатно, без регистрации сложной.
- **GitHub Pages** — если есть аккаунт GitHub.
- **Свой сервер/поддомен**, например `config.tomaa.ru` — если есть хостинг.

После загрузки проверьте, что открывается `https://ВАШ-АДРЕС/index.html`.

### Шаг 2. Вставить на страницу Tilda
1. На нужной странице добавьте блок **«Другое → HTML»** (код блока **T123**).
2. Откройте файл `tilda-embed.html`, скопируйте его содержимое в этот блок.
3. Замените `АДРЕС-КОНФИГУРАТОРА` на ваш адрес из шага 1.
4. Опубликуйте страницу.

Высота конфигуратора подстроится автоматически (он сам сообщает свою высоту окну Tilda).

> **Альтернатива без iframe.** Можно вставить содержимое `index.html` (без тегов `<html>/<body>`)
> прямо в HTML-блок, а картинки залить в Tilda File Manager и прописать их общий адрес в
> строке `const BASE = "assets/";` внутри `index.html`. Это дольше (≈19 файлов) и
> рекомендуется только если нельзя пользоваться внешним хостингом.

## 4. Что нужно от вас, чтобы довести до финала

Каталог Tilda подключён полностью: основы (кожа/бархат, цвета, узкий/широкий, один/два оборота)
и шармы — с реальными названиями, ценами и UID. Кнопка «Оформить заказ» кладёт в корзину Tilda
и основу, и все шармы. Осталось по визуалу:

1. **Развёрнутые фото основ.** Сейчас реальное фото есть только для узкого 10мм (чёрного).
   Формы 15мм и «на два оборота» собраны из него (растяжение/копия), а **цвета** — программная
   перекраска по яркости (для светлых кож и бархата выходит приблизительно). Чтобы было точно,
   пришлите развёрнутые фото (вид сверху, белый фон): узкий 15мм, узкий 2 оборота, широкий
   2 оборота — и, по возможности, реальные цвета/бархат. Заменим и перегенерим.

2. **Проверка корзины на tomaa.ru.** Добавление в корзину (`tcart__addProduct`) надо
   протестировать на живой странице — на превью корзины Tilda нет, поэтому состав уходит в
   событие `tomaa:order`. Если их версия корзины ждёт другие поля — поправим по факту.

## 5. Как настраивать (для разработчика)

Всё управление — вверху `<script>` в `index.html`:

- **Каталог шармов** — массив `CHARMS_FULL` (74 шарма из выгрузки): `id` (Tilda UID), `n`
  (название), `c` (категории), `loc` (латинский id, если есть локальная картинка для наложения),
  `ph` (фото с tildacdn для превью), `v` (варианты `покрытие-ширина` → UID и цена). Палитра
  фильтруется по категориям (`CATEGORIES`). Накладываются только шармы с `loc` (сейчас 8 из
  серии «Удача»); у остальных есть превью и они помечены «скоро» — добавьте им локальные
  фронтальные картинки (`charms/{gold,silver}/{loc}.webp` + превью) и проставьте `loc`, чтобы
  включить примерку. Цена шарма берётся по покрытию и ширине основы автоматически.
- **Корзина Tilda** — функция `addToTildaCart()` добавляет шармы через `tcart__addProduct`
  по их UID; срабатывает на странице tomaa.ru. Основу нужно добавить туда же, когда будут UID.
- **Добавить/убрать шарм** — массив `CHARMS` (плюс положить файлы в `assets/charms/gold|silver/`
  и `assets/charms/preview/gold|silver/`).
- **Каталог основ** — объект `BASES`: материал (кожа/бархат) → цвета (`id/name/hex`) и Tilda UID
  по геометрии и цвету. Геометрия и её цена — в `VARIANTS` (узкий/широкий × один/два оборота,
  1290/1990 ₽). Картинки основ — `bracelets/{геометрия}-{материал}-{цвет}.webp`, генерируются
  скриптом `recolor_bases.py` (перекраска чёрной формы). После замены фото запускать его.
- **Уровни шарма (для двойных основ)** — массив `charm.levels` у варианта: каждый уровень
  имеет `id`, `label` («Верхний»/«Нижний») и `top` (центр уровня в % от высоты картинки).
  У одинарных — один уровень, селектор уровня скрыт автоматически.
- **Подгонка положения/размера шарма** — `charm.left` (центр по горизонтали, %) и
  `charm.width` (размер шарма в % от ширины браслета); вертикаль задаётся уровнями.
- **Максимум шармов и их плотность** — `MAX_CHARMS` (по умолчанию 5 на ремешок) и `CHARM_STEP`
  (шаг между шармами как доля их ширины; меньше → плотнее, сейчас 0.92).
- **Состав заказа** — в `onOrder()` объект `order` содержит `charms` — массив надетых шармов
  `{level, charm, plating}` — плюс `count` и `price`; и событие `tomaa:order` для интеграции.
- **Подключение кнопки заказа** — функция `onOrder()`: туда приходит готовый объект `order`,
  и есть событие `tomaa:order` для интеграции с корзиной/формой.
- **Адрес картинок при хостинге** — строка `const BASE = "assets/";` (можно указать полный URL).
- **Оформление** подогнано под сайт tomaa.ru: шрифт **Montserrat** (подключается с Google
  Fonts; на Tilda он уже загружен), чёрно-белая палитра, заголовки UPPERCASE, плоские кнопки
  с радиусом 5–6 px. Все цвета — в CSS-переменных блока `:root` (`--ink`, `--line`, `--accent`,
  `--radius` и т.д.), менять там.

Артикул собирается автоматически, например: `TOMAA-SINGLE-10-GOLD-DRAGON`.
